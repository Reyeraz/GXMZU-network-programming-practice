实验报告：前后端联调与核心功能测试


（1）CORS 跨域全局配置

后端在 WebConfig.java 中配置了全局 CORS，允许前端 localhost:5173 和 localhost:3000 两个来源访问，支持 GET、POST、PUT、DELETE、OPTIONS 五种请求方法，允许所有请求头。配置代码如下：

registry.addMapping("/**")
    .allowedOrigins("http://localhost:5173", "http://localhost:3000")
    .allowedMethods("GET", "POST", "PUT", "DELETE", "OPTIONS")
    .allowedHeaders("*");

同时在 JwtInterceptor 中放行 OPTIONS 预检请求，确保浏览器跨域预检不会被拦截器拒绝：

if ("OPTIONS".equalsIgnoreCase(request.getMethod())) {
    return true;
}

这样所有 /products/** 和 /cart/** 接口都允许前端跨域请求，商品接口属于公开接口直接放行，购物车接口需要 JWT 认证但 CORS 不受影响。


（2）前后端联调实现

前端请求工具对接后端真实接口的工作过程：

1. Vite 代理配置：vite.config.ts 中配置 /api 代理，将前端请求转发到 http://localhost:8080，并去掉 /api 前缀。例如前端请求 /api/products/page 会被转发为 http://localhost:8080/products/page。

2. Axios 请求拦截器：request.js 创建 Axios 实例，baseURL 设为 /api，请求拦截器自动从 localStorage 读取 Token 并添加 Authorization: Bearer <token> 请求头，白名单内的接口（如商品浏览）不添加 Token。

3. 商品列表查询：ProductList.vue 页面加载时调用 productAPI.getProductsByPage()，发送 GET 请求到 /products/page，携带 page、pageSize、category、keyword 参数。后端 ProductController 接收请求，ProductService 使用 LambdaQueryWrapper 构建查询条件，MyBatis-Plus 分页插件执行分页 SQL，返回 PageResult 包含 items 列表和 total 总数。

4. 商品详情查询：ProductDetail.vue 通过路由参数 id 调用 productAPI.getProductById()，发送 GET 请求到 /products/{id}，后端通过 selectById 查询单个商品返回。

5. 购物车添加商品：ProductList.vue 和 ProductDetail.vue 中点击"加入购物车"按钮，先检查用户登录状态，调用 cartStore.addToCart()，发送 POST 请求到 /cart，携带 productId 和 quantity。后端 CartController 接收请求，CartService 检查购物车中是否已有该商品，有则更新数量，无则新增记录。

6. 购物车数量修改：Cart.vue 中使用 el-input-number 组件，数量变化时调用 cartStore.updateQuantity()，发送 PUT 请求到 /cart/{cartId}，携带 newQuantity。后端 CartService 更新对应购物车记录的数量。

7. 购物车状态同步：每次操作（添加、修改数量、修改勾选状态、删除）成功后，cartStore 都会重新调用 loadCart() 从后端获取最新购物车列表，确保前端状态与后端数据一致。


（3）联调中遇到的问题及解决方案

问题1：数据格式不一致
现象：前端期望的字段名与后端返回的不一致，例如后端返回 productName 但前端使用 name。
解决：在 cartStore.loadCart() 中做字段映射转换，将后端的 productName 映射为前端的 name，cartId、productId、price、quantity、selected 等字段保持一致。

问题2：请求参数错误
现象：添加购物车时后端返回参数校验失败。
解决：确保前端发送的请求体格式与后端 DTO 一致，添加购物车使用 {productId, quantity}，更新数量使用 {newQuantity}，更新勾选使用 {selected}。

问题3：跨域失效
现象：浏览器控制台报 CORS 错误。
解决：WebConfig 中已配置 CORS，但需要确保 JwtInterceptor 放行 OPTIONS 预检请求，否则预检被拦截会导致后续实际请求失败。在 preHandle() 开头添加 OPTIONS 方法判断并直接返回 true。

问题4：购物车状态同步异常
现象：添加商品后购物车列表未更新。
解决：cartStore 的每个异步操作（addToCart、updateQuantity、updateSelected、removeCartItem 等）在后端返回成功后都调用 await loadCart() 重新加载最新数据，确保前端状态与后端一致。使用 async/await 确保操作顺序正确。

问题5：Token 过期处理
现象：用户长时间未操作后请求返回 401。
解决：Axios 响应拦截器捕获 401 状态码时，清除 localStorage 中的 token、userId、username，提示用户重新登录，跳转到登录页面。
