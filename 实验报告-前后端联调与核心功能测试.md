实验报告：电商前后端联调与核心功能测试


实验任务与要求

本次实验的任务与要求如下：

1. 在电商后端工程中配置 CORS 跨域全局配置，确保所有商品/购物车接口允许前端请求。

2. 将前端请求工具对接后端真实接口，实现商品列表/查询/分页、购物车商品添加/数量修改的前后端联调。

3. 排查联调中的问题（数据格式不一致、请求参数错误、跨域失效、购物车状态同步异常等），逐一解决。

4. 完成电商核心功能测试，确保前后端交互正常，数据同步一致。


实验工具

1. 开发IDE：IntelliJ IDEA 2025.2（后端Java开发、Maven构建）

2. 前端开发：VS Code + Vite开发服务器（Vue 3 + TypeScript）

3. 后端框架：Spring Boot 3.4.4 + Java 21 + MyBatis-Plus

4. 前端框架：Vue 3 + Pinia + Element Plus + Axios

5. 数据库：MySQL（远程 p-c12.polars.cc:54237，数据库 user_db）

6. 版本控制：Git + GitHub + Gitee

7. 接口测试：IntelliJ HTTP Client（.http 文件）

8. 浏览器：Chrome（开发者工具调试网络请求和控制台）


实验分析与设计

1. 跨域问题分析：前端运行在 localhost:5173，后端运行在 localhost:8080，浏览器同源策略会阻止跨域请求。解决方案是在后端 WebConfig 中配置 CORS 全局允许前端来源，同时在 JWT 拦截器中放行 OPTIONS 预检请求。

2. 前后端联调设计：前端通过 Vite 代理将 /api 前缀请求转发到后端，Axios 拦截器自动注入 JWT Token，后端拦截器验证 Token 后将 userId 存入请求属性，Controller 从请求属性获取当前用户。

3. 数据同步设计：购物车每次操作（添加、修改、删除）成功后，前端重新调用 loadCart() 从后端获取最新数据，确保前后端状态一致。

4. 错误处理设计：Axios 响应拦截器统一处理 HTTP 错误码，401 清除 Token 跳转登录，403 提示无权限，500 提示服务器错误。


实验过程

一、CORS 跨域全局配置

在 WebConfig.java 中添加 CORS 配置，允许前端来源访问所有接口：

代码：addCorsMappings 方法配置 allowedOrigins 为 http://localhost:5173 和 http://localhost:3000，allowedMethods 为 GET、POST、PUT、DELETE、OPTIONS，allowedHeaders 为通配符 *。

在 JwtInterceptor.java 的 preHandle() 方法开头添加 OPTIONS 预检请求放行逻辑，判断请求方法为 OPTIONS 时直接返回 true，不进行 Token 验证。

测试结果：前端请求商品接口和购物车接口均正常返回数据，浏览器控制台无 CORS 错误。

二、前端请求工具对接后端

1. Vite 代理配置：vite.config.ts 中 server.proxy 配置 /api 代理到 http://localhost:8080，rewrite 去掉 /api 前缀。

2. Axios 实例创建：request.js 中创建 baseURL 为 /api 的 Axios 实例，设置超时时间 10 秒。

3. 请求拦截器：从 localStorage 读取 Token，非白名单接口自动添加 Authorization: Bearer <token> 请求头。

4. 响应拦截器：成功响应直接返回 response.data，错误响应根据状态码弹出对应提示，401 时清除本地存储并跳转登录页。

5. API 方法封装：api.ts 中封装 productAPI（getAllProducts、getProductById、searchProducts、getProductsByPage、getCategories）、userAPI（register、login）、cartAPI（getCartList、addToCart、updateQuantity、updateSelected、batchUpdateSelected、deleteCartItem、deleteSelected）等方法。

测试结果：商品列表页加载时正确调用 getProductsByPage 接口，返回分页数据渲染商品卡片。

三、购物车联调

1. 购物车 Store：cart.ts 中定义 useCartStore，包含 items 响应式数组和 totalQuantity、totalPrice、selectedCount 计算属性。

2. 添加购物车：ProductList.vue 和 ProductDetail.vue 中点击"加入购物车"按钮，先检查登录状态，调用 cartStore.addToCart()，发送 POST /cart 请求，成功后重新加载购物车列表。

3. 数量修改：Cart.vue 中 el-input-number 组件的 change 事件触发 cartStore.updateQuantity()，发送 PUT /cart/{cartId} 请求，成功后重新加载。

4. 勾选状态：单个勾选调用 cartStore.updateSelected()，全选调用 cartStore.batchUpdateSelected()，成功后重新加载。

5. 删除商品：单个删除调用 cartStore.removeCartItem()，批量删除调用 cartStore.deleteSelected()，成功后重新加载。

测试结果：购物车添加商品后列表自动更新，数量修改后总价实时变化，勾选状态切换正常，删除操作后列表刷新。

四、联调问题排查与解决

问题1：数据格式不一致
现象：后端 CartItemVO 返回 productName 字段，前端 CartItem 接口使用 name 字段。
解决：在 cartStore.loadCart() 中使用 map 做字段映射，item.productName 映射为 name。

问题2：请求参数格式错误
现象：添加购物车返回 400 参数校验失败。
解决：检查后端 AddToCartRequest DTO 字段为 productId 和 quantity，前端发送 {productId: product.id, quantity: 1} 格式匹配。

问题3：跨域预检失败
现象：POST/PUT 请求在浏览器 Network 面板显示 CORS 错误。
解决：JwtInterceptor 中添加 OPTIONS 方法判断，预检请求直接放行不走 Token 验证逻辑。

问题4：购物车状态不同步
现象：快速连续操作购物车后，前端显示数量与数据库不一致。
解决：每个异步操作使用 await 等待完成后再调用 loadCart()，避免并发操作导致数据覆盖。

问题5：Token 过期无提示
现象：用户长时间未操作后请求静默失败。
解决：响应拦截器捕获 401 状态码，ElMessage.error 提示"登录已过期"，清除 localStorage 并跳转登录页。


实验总结

1. 实验收获：通过本次实验，掌握了前后端分离架构下的 CORS 跨域配置方法、Axios 请求拦截器和响应拦截器的使用、Pinia 状态管理与后端数据同步的实现方式、前后端联调中常见问题的排查思路。

2. 遇到的问题及解决：主要问题集中在跨域配置（OPTIONS 预检放行）、数据格式映射（前后端字段名不一致）、状态同步（异步操作顺序控制）三个方面。通过浏览器开发者工具的 Network 面板和 Console 面板定位问题，逐一修改解决。

3. 注意事项：CORS 配置不仅要设置 allowedOrigins，还要处理 OPTIONS 预检请求；前后端联调时要统一接口文档，确保请求参数和响应格式一致；购物车等需要实时同步的功能，每次操作后都要重新获取最新数据，避免本地状态与数据库不一致。

4. 改进方向：可以引入 TypeScript 接口定义统一前后端数据类型，减少字段映射错误；可以添加请求重试机制提高网络容错性；可以使用 WebSocket 实现购物车实时同步，替代轮询方式。
