### 完成内容

#### 自定义异常体系
- **新增exception包**，建立三层异常继承结构：
  - `BusinessException`（基类，extends RuntimeException，含 int code 表示HTTP状态码）
  - `BadRequestException`（code=400）— 业务校验失败（库存不足/商品下架/购物车空等）
  - `ForbiddenException`（code=403）— 越权操作他人资源
  - `ResourceNotFoundException`（code=404）— 资源不存在（订单/购物车项等）

#### GlobalExceptionHandler改造
- 返回类型从 `ApiResponse<?>` 改为 `ResponseEntity<ApiResponse<?>>`，按异常类型返回对应HTTP状态码：
  - `BadRequestException` → 400
  - `ForbiddenException` → 403
  - `ResourceNotFoundException` → 404
  - `MethodArgumentNotValidException` → 400（原来200）
  - `RuntimeException` → 500（兜底）
